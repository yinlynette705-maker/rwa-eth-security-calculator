# RWA 安全市值推算器：五类模型说明

本文档记录当前版本使用的五类安全模型，包括模型含义、普通用户解释、计算方式、默认参数和参数选择理由。

## 统一输出

页面的核心目标是估算：当 ETH 链上承载一定规模的 RWA 时，ETH 应该对应多大的市值和币价，才具备足够的经济安全水平。

统一计算：

```text
所需 ETH 市值 = RWA 总规模 × 模型倍数
全供应口径 ETH 价格 = 所需 ETH 市值 / ETH 当前供应量
流动 ETH 供应量 = ETH 当前供应量 × (1 - ETH 当前质押比例)
流动口径 ETH 价格 = 所需 ETH 市值 / 流动 ETH 供应量
约为当前价格 = 流动口径 ETH 价格 / ETH 当前价格
```

其中：

- `RWA 总规模`：页面滑块控制的 ETH 链上 RWA 总规模。
- `ETH 当前供应量`：手动维护的数据，暂不考虑通胀、通缩、燃烧和发行变化。
- `ETH 当前质押比例`：手动维护的数据，用于估算未质押的流动 ETH 供应量。
- `ETH 当前价格`：手动维护的数据，用于计算流动口径价格约为当前价格的多少倍。
- `ETH 当前市值`：手动维护的数据，用于展示当前 ETH 的市场规模参考。

两个价格口径的区别：

- `全供应口径 ETH 价格`：把所有 ETH 供应量都视为价格计算基数，适合做最简单的静态市值换算。
- `流动口径 ETH 价格`：把已质押 ETH 从供应量中扣除，只用未质押供应量计算理论价格。这个口径用于反映质押率上升后，剩余流动供应量变少，达到同样所需市值时价格要求会更高。

注意：流动口径仍然是简化估算。它没有进一步模拟解质押周期、交易所订单簿深度、长期持有者行为、衍生品流动性或市场情绪。

注意：这是启发式压力测试，不是严格的密码经济学证明，也不是投资建议。

## 1. 短期重组 / Reorg

### 含义

短期重组指链上已经确认不久的交易，在短时间内被回滚或改写。

### 普通用户解释

对 RWA 来说，它类似于：一笔资产结算刚刚显示完成，但短时间后链上记录被翻回去，导致结算结果不再成立。

### 建议公式

```text
所需 ETH 市值 = RWA 总规模 × 短期可攻击价值比例 × 安全冗余倍数 / 短期攻击资本效率
```

### 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 短期可攻击价值比例 | 5% | 假设短窗口内真正可被重组影响的 RWA 不是全部规模，而是其中一小部分结算流量或待确认价值 |
| 安全冗余倍数 | 3x | 给不确定性、流动性冲击和攻击协调成本留出缓冲 |
| 短期攻击资本效率 | 15% | 假设攻击者每投入 1 美元攻击资本，可能影响约 0.15 美元等价的 ETH 经济安全约束 |

由此得到：

```text
5% × 3 / 15% = 1.0
```

### 当前页面计算方式

```text
所需 ETH 市值 = RWA 总规模 × 1.0
```

也就是：

```text
模型倍数 = 1.0
```

### 参数选择理由

短期重组通常不是为了长期控制整条链，而是为了在短窗口内改变某些交易结果。因此第一版采用一个直观、保守但不过度复杂的口径：ETH 的经济安全基础至少要和链上 RWA 规模处于同一量级。

这个模型的重点不是精确计算每一次重组成本，而是提醒用户：当 RWA 规模接近或超过底层链经济规模时，短期结算回滚风险会变得更值得关注。

## 2. 最终性延迟 / 阻止

### 含义

最终性延迟或阻止指交易没有被回滚，但迟迟无法获得最终确认。

### 普通用户解释

对 RWA 来说，它类似于：资产交割、赎回、清算或兑付流程被卡住，不能顺利完成。

### 建议公式

```text
所需 ETH 市值 = RWA 总规模 × 每日业务影响比例 × 延迟天数 × 安全冗余倍数 / 最终性攻击约束比例
```

### 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 每日业务影响比例 | 0.25% | 假设最终性被延迟时，每天直接受影响的是一小部分结算、赎回、清算或交割流程 |
| 延迟天数 | 3 天 | 用一个中短期中断窗口衡量最终性异常带来的业务影响 |
| 安全冗余倍数 | 5x | 最终性问题会放大信任冲击，因此使用高于短期重组的冗余 |
| 最终性攻击约束比例 | 33% | PoS 中影响最终性的常见门槛直觉，不等同于长期控制整条链 |

由此得到：

```text
0.25% × 3 × 5 / 33% ≈ 0.114
```

### 当前页面计算方式

```text
所需 ETH 市值 = RWA 总规模 × 0.114
```

也就是：

```text
模型倍数 = 0.114
```

### 参数选择理由

最终性延迟的直接影响通常不是“全部 RWA 归零”，而是部分业务流程在一段时间内无法完成。因此这里用“每日影响比例 × 延迟天数”来估算风险暴露，再乘以安全冗余。

`33%` 来自 PoS 最终性相关攻击的常见门槛直觉：攻击者不一定需要长期掌握多数，但达到足够比例后可能影响最终性。第一版采用 `0.114 × RWA`，使它成为一个偏运行中断和结算卡顿的压力测试，而不是长期接管模型。

## 3. 51%+ 长期攻击

### 含义

51%+ 长期攻击指攻击者长期掌握共识主导权，从而影响链上裁决和交易排序的可信度。

### 普通用户解释

对 RWA 来说，它类似于：承载资产记录和结算的基础账本被长期控制，市场不再信任这条链能公正处理资产状态。

### 建议公式

```text
所需 ETH 市值 = RWA 总规模 × 系统性暴露比例 × 安全冗余倍数 / 长期控制比例
```

### 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 系统性暴露比例 | 50% | 假设长期共识攻击会影响较大比例的 RWA 信任基础，但不简单假设 100% 直接损失 |
| 安全冗余倍数 | 4x | 长期接管会冲击链的整体可信度，因此需要较高安全冗余 |
| 长期控制比例 | 51% | 使用普通用户熟悉的长期多数控制门槛作为抽象近似 |

由此得到：

```text
50% × 4 / 51% ≈ 3.92
```

### 当前页面计算方式

```text
所需 ETH 市值 = RWA 总规模 × 3.92
```

也就是：

```text
模型倍数 = 3.92
```

### 参数选择理由

长期共识攻击的破坏不只影响单笔交易，还会影响整条链作为结算层的可信度。因此这里假设有较高比例的 RWA 会受到系统性影响，并使用较高安全冗余。

`51%` 是用户较容易理解的长期控制门槛。实际 PoS 攻击机制比这个更复杂，但作为普通用户可读的压力测试，`3.92 × RWA` 能表达一个核心判断：长期共识安全需要远高于被承载资产的局部风险敞口。

## 4. 经济攻击 / 治理攻击

### 含义

经济攻击或治理攻击不是直接攻击 ETH 共识本身，而是攻击围绕 RWA 运转的关键依赖。

### 普通用户解释

攻击者可能攻击价格、流动性、治理、预言机、桥、托管、清算机制或其他外部系统。链本身可能还在运行，但资产系统已经受到破坏。

### 建议公式

```text
所需 ETH 市值 = RWA 总规模 × 外围依赖暴露比例 × 安全冗余倍数 / 经济攻击资本效率
```

### 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 外围依赖暴露比例 | 20% | 假设 RWA 系统中有相当一部分风险来自预言机、托管、治理、流动性、桥等外围环节 |
| 安全冗余倍数 | 3x | 为复杂依赖、连锁反应和市场信心冲击留出缓冲 |
| 经济攻击资本效率 | 10% | 假设攻击者可能用较少资本撬动较大系统性影响，因此资本效率参数设得较低 |

由此得到：

```text
20% × 3 / 10% = 6.0
```

### 当前页面计算方式

```text
所需 ETH 市值 = RWA 总规模 × 6.0
```

也就是：

```text
模型倍数 = 6.0
```

### 参数选择理由

RWA 不只依赖底层链安全，也依赖链外资产映射、预言机、托管、赎回、治理和流动性等系统。很多攻击不需要直接控制共识，只要能用较少资本撬动关键依赖，就可能造成较大影响。

因此这个模型给出当前版本中较高的倍数。它提醒用户：即使 ETH 共识很强，RWA 系统的外围环节也可能成为更脆弱的入口。

## 5. 安全缓冲倍数

### 含义

安全缓冲倍数不拆分具体攻击路径，而是直接用一个抽象倍数衡量 ETH 的经济安全垫是否足够厚。

### 普通用户解释

它类似于问：ETH 这张底层“安全垫”够不够厚，能不能承载这么大规模的链上 RWA。

### 建议公式

```text
所需 ETH 市值 = RWA 总规模 × 安全缓冲倍数
```

### 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 安全缓冲倍数 | 5x | 用一个简单、易理解的倍数表达底层链经济安全垫应明显大于承载资产规模 |

由此得到：

```text
5.0
```

### 当前页面计算方式

```text
所需 ETH 市值 = RWA 总规模 × 5.0
```

也就是：

```text
模型倍数 = 5.0
```

### 参数选择理由

这个模型用于给普通用户一个最直观的参考，不要求理解具体攻击机制。`5 × RWA` 表达的是：底层链的经济规模应当明显大于其承载的真实资产规模，而不是只和 RWA 规模勉强相等。

它也可以作为其他模型之间的中位参考：比短期重组更保守，比经济攻击模型略低，适合做整体安全缓冲的直觉锚点。

## 当前默认模型倍数汇总

| 模型 | 默认倍数 | 含义 |
|---|---:|---|
| 短期重组 / Reorg | 1.0x | 所需 ETH 市值与 RWA 规模同量级 |
| 最终性延迟 / 阻止 | 0.114x | 估算结算卡顿和最终性中断的风险暴露 |
| 51%+ 长期攻击 | 3.92x | 估算长期控制共识所需的经济安全规模 |
| 经济攻击 / 治理攻击 | 6.0x | 估算外围依赖被经济方式攻击时的安全要求 |
| 安全缓冲倍数 | 5.0x | 抽象衡量 ETH 经济安全垫是否足够厚 |

## 后续可调整项

后续如果需要提高严谨度，可以把每个模型的参数从固定倍数拆成可配置项，例如：

- 短期可攻击 RWA 比例
- 安全冗余倍数
- 攻击资本效率
- 延迟天数
- 每日业务影响比例
- 外围依赖暴露比例
- 长期共识控制比例

当前版本先保持前端简单：用户只调整 RWA 总规模，其他模型参数使用默认值。
